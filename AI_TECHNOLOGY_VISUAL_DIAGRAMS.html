How Text Guides Generation

At each of the 50 steps, the text prompt influences which noise to remove:

  • Text Embeddings are computed once at the start
  • U-Net uses these embeddings at every denoising step
  • Guidance Scale (7.5) controls how strictly to follow the prompt
  • Higher guidance = more literal interpretation
  • Lower guidance = more creative freedom

Component Details

CLIP Text Encoder

Purpose: Convert text to numerical embeddings

Training: 400M image-text pairs

Output: 768-dimensional vector representing semantic meaning

U-Net

Purpose: Predict noise at each step

Architecture: Encoder-decoder with skip connections

Input: Noisy latent + text embeddings + timestep

Output: Predicted noise to remove

VAE (Variational Autoencoder)

Purpose: Compress/decompress images

Compression: 512x512 → 64x64 (16x smaller)

Benefit: Faster processing, less memory

🔐 Security Architecture

✅ CORRECT

# Backend (.env file) GEMINI_API_KEY=secret_key_here # Python code from dotenv import load_dotenv load_dotenv() api_key = os.getenv("GEMINI_API_KEY")

✓ API key on backend only
✓ Never exposed to browser
✓ Secure environment variables

❌ WRONG

// Frontend JavaScript const API_KEY = "secret_key_here"; fetch(url, { headers: { 'Authorization': API_KEY } });

✗ API key in frontend code
✗ Visible in browser
✗ Anyone can steal it

Security Features

  • API Key Protection: Stored in .env file, never exposed to frontend
  • CORS Configuration: Controls which domains can access the API
  • Input Validation: Checks prompt length and content
  • Error Handling: Doesn't expose internal details
  • Rate Limiting: Can be added to prevent abuse
